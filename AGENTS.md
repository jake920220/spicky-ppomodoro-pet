# AGENTS.md

## 프로젝트 개요

Spiky(스피키)는 macOS와 Windows에서 동작하는 가벼운 데스크톱 펫 타이머 앱입니다.

스피키는 바탕화면 위에 작은 오버레이 캐릭터로 표시되며, 약간의 흔들거리는 애니메이션과 함께 걸어다니는 시각적 효과를 보엽줍니다. 클릭 시 가벼운 상호작용이 있습니다.
준비된 투명 PNG 3장과 MP3 음원 2개를 사용합니다.

## 현재 범위

반드시 포함:

- Tauri 2
- 프론트엔드는 우선 Vanilla TypeScript 사용
- 데스크톱 전용: macOS / Windows
- 기본 상태는 image_1 사용
- 클릭 상호작용 상태는 image_2 사용, 클릭 시 MP3 1 재생
- 타이머 종료 알림 상태는 image_3 사용, 타이머 종료 시 MP3 2 재생
- 뽀모도로 스타일 타이머

현재 제외:

- LLM 연동
- 온라인 API
- 백엔드
- 로그인/인증
- 클라우드 동기화
- 분석 도구
- 모바일 지원

## 기술 원칙

- 앱은 가볍고 단순하게 유지한다.
- 꼭 필요한 경우가 아니면 Rust 쪽 구현은 최소화한다.
- 타이머 로직과 캐릭터 상태 로직은 분리한다.
- 오디오 재생 로직은 타이머 로직과 분리한다.
- 에셋 경로는 한곳에서 관리한다.
- 불필요한 의존성은 추가하지 않는다.
- 작은 단위로 변경하고 검토 가능하게 유지한다.
- 구현 후에는 어떤 파일이 왜 바뀌었는지 짧게 설명한다.
- 구현 후에는 항상 수동 검증 체크리스트를 제공한다.

## 상태 모델

### 시각 상태

- default -> image_1
- clicked -> image_2
- timer_finished -> image_3

### 타이머 상태

- idle
- running
- paused
- finished

## UX 원칙

- 스피키 클릭 반응은 즉각적이어야 한다.
- 타이머 종료 시 시각적 알림과 오디오 알림이 모두 있어야 한다.
- 조작 UI는 작고 단순해야 한다.
- v1에서는 화려함보다 안정성이 더 중요하다.

## 아키텍처 원칙

다음 관심사를 분리한다:

- Tauri 창/쉘 동작
- 타이머 도메인 로직
- 스피키 시각 상태
- 오디오 재생
- 에셋 경로 관리

## 구현 우선순위

1. 스피키 기본 표시
2. 클릭 시 상태 전환
3. 타이머 시작/정지/완료
4. 완료 시 image_3 전환
5. 완료 시 MP3 재생
