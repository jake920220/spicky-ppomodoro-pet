# Spiky Ppomodoro pet

> 스피키 뽀모도로 펫

Spiky Ppomodoro pet(스피키 뽀모도로 펫)은 트릭컬의 밈 캐릭터인 스피키에서 모티브를 가져온 비공식 팬 프로젝트입니다.
상업적 목적이나 수익화를 전제로 한 프로젝트가 아니라, 비상업적·비영리적으로 개발자 개인의 팬심에서 출발한 사이드 프로젝트라는 점을 분명히 두고 개발하고 있습니다.

이 저장소는 원작 IP에 대한 어떠한 권리도 주장하지 않습니다.
트릭컬, 스피키 캐릭터 및 관련 명칭·세계관·원저작물에 대한 권리는 해당 권리자에게 귀속되며, 본 프로젝트는 개인적인 팬 활동의 연장선에서 개발되는 비공식 구현입니다.

macOS와 Windows 데스크톱 위를 돌아다니는 작은 펫처럼 동작하면서, 동시에 가볍고 직관적인 뽀모도로 타이머 경험을 제공합니다.

이 프로젝트의 목표는 단순한 "타이머 앱"이 아니라, 바탕화면 위에서 살아 있는 듯한 캐릭터가 사용자의 집중 시간을 함께 지켜주는 경험을 만드는 것입니다. 준비된 PNG 3장과 MP3 2개를 활용해 기본 상태, 클릭 반응, 타이머 종료 알림을 구성하고, 데스크톱 펫다운 움직임과 상호작용을 점진적으로 다듬고 있습니다.

## 핵심 기능

- macOS / Windows 데스크톱 오버레이 펫
- 투명 창 기반의 always-on-top 캐릭터 표시
- 랜덤 이동과 클릭 반응을 가진 데스크톱 펫 상호작용
- 뽀모도로 스타일 타이머
- 타이머 종료 시 시각 상태 전환 + 오디오 알림
- 로컬 전용 동작, 서버/로그인/클라우드 의존성 없음

## 실행 방법

배포 파일은 GitHub Releases 페이지에 버전별로 업로드하는 것을 기준으로 합니다.

1. Releases 페이지에서 운영체제에 맞는 설치 파일을 내려받습니다.
2. macOS 사용자는 `.dmg`를 열고 앱을 `Applications`로 옮긴 뒤 실행합니다.
3. Windows 사용자는 `-setup.exe` 또는 `.msi` 설치 파일을 실행해 설치합니다.

현재는 무료 unsigned 배포를 기준으로 준비하고 있어서, 처음 실행할 때 운영체제 경고가 나올 수 있습니다.

### macOS에서 실행할 때

- 브라우저에서 내려받은 앱이라는 안내가 뜰 수 있습니다.
- 이 경우 `시스템 설정 > 개인정보 보호 및 보안`에서 `그래도 열기`를 선택하면 실행할 수 있습니다.

### Windows에서 실행할 때

- SmartScreen 경고가 뜰 수 있습니다.
- 이 경우 `추가 정보`를 누른 뒤 `실행`을 선택하면 설치를 계속할 수 있습니다.

## 왜 이런 기술 스택을 선택했는가

### Tauri 2

Spiky는 데스크톱 펫이기 때문에 "항상 위에 떠 있는 가벼운 창", "투명한 오버레이", "윈도우 위치 제어"가 핵심입니다.
Tauri 2는 이 요구사항을 만족하면서도 Electron보다 훨씬 가볍고, 데스크톱 유틸리티 성격의 앱을 만들기에 적합합니다.

### Vanilla TypeScript

초기 버전에서는 거대한 UI 프레임워크보다, 상태 흐름과 상호작용을 직접 통제하는 편이 더 중요하다고 판단했습니다.
Spiky는 복잡한 대시보드가 아니라 작은 오버레이 앱이기 때문에, 지금 단계에서는 Vanilla TypeScript가 가장 단순하고 유지보수하기 쉬운 선택입니다.

### pnpm

빠르고 일관된 의존성 관리가 필요했고, 프로젝트 규모에 비해 불필요한 설치 비용을 줄이기 위해 `pnpm`을 선택했습니다.

## 개발 방식

이 프로젝트는 **Codex + GSD(Get Shit Done)** 를 사용해 **SDD(Spec Driven Development)** 방식으로 진행됩니다.

즉, "일단 만들고 나중에 정리"하는 방식이 아니라:

1. 아이디어를 문서화하고
2. 요구사항을 정의하고
3. Phase 단위로 계획을 세우고
4. 실행 후 검증하고
5. 다음 Phase로 넘어가는 흐름을 유지합니다

이 과정은 `.planning/` 아래의 문서로 누적되며, 구현과 의사결정이 분리되지 않도록 설계되어 있습니다.

## 실제 개발 예시

이 프로젝트에서는 아래와 같은 흐름으로 작업이 진행되었습니다.

```bash
$gsd-map-codebase
$gsd-new-project --auto idea.md
$gsd-discuss-phase 1
$gsd-plan-phase 1
$gsd-execute-phase 1
```

그 결과 다음과 같은 문서가 함께 생성됩니다.

- `.planning/PROJECT.md`
- `.planning/REQUIREMENTS.md`
- `.planning/ROADMAP.md`
- `.planning/STATE.md`
- `.planning/phases/01-overlay-shell-foundation/01-CONTEXT.md`
- `.planning/phases/01-overlay-shell-foundation/01-01-PLAN.md`
- `.planning/phases/01-overlay-shell-foundation/01-VERIFICATION.md`

이 문서들은 단순 기록이 아니라, 실제 구현과 검증의 기준점 역할을 합니다.

## 왜 SDD를 선택했는가

Spiky Ppomodoro pet 같은 프로젝트는 겉보기엔 단순해 보여도, 실제로는 다음과 같은 문제가 쉽게 생깁니다.

- UI 감각 수정이 반복되며 방향이 흔들림
- "지금 뭐가 완료된 건지" 기준이 모호해짐
- 작은 데스크톱 상호작용이 누적되면서 상태 로직이 꼬임
- 타이머, 캐릭터 상태, 오디오, 창 이동이 서로 강하게 얽힘

SDD는 이런 문제를 줄여줍니다.

### SDD의 장점

- 구현 전에 범위를 고정해 불필요한 확장을 줄일 수 있음
- Phase 단위로 목표와 완료 조건이 명확해짐
- 요구사항과 실제 코드 변경이 연결됨
- "왜 이렇게 만들었는지"가 문서에 남아 다음 세션에서도 이어짐
- Codex 세션을 분리해도 계획/구현/리뷰 흐름을 유지할 수 있음

### 이 프로젝트에서 특히 중요한 이유

Spiky Ppomodoro pet은 앞으로도 단순한 타이머를 넘어서 "작은 데스크톱 캐릭터 경험"을 다듬어 가는 프로젝트입니다.
이럴수록 감성적인 UI 결정과 기술적인 제약을 함께 관리해야 하므로, SDD 방식이 훨씬 안정적입니다.

## 현재 개발 방향

현재는 다음 우선순위로 진행 중입니다.

1. 데스크톱 펫으로서 자연스러운 기본 표시
2. 클릭 반응과 이동 상호작용 개선
3. 타이머 흐름과 완료 피드백 정교화
4. macOS / Windows 환경에서의 안정성 강화

## 저작권 및 고지

- 본 프로젝트는 비공식 팬 프로젝트입니다.
- 상업적 이용이나 수익화를 목적으로 하지 않습니다.
- 트릭컬, 스피키 캐릭터, 관련 명칭 및 원저작물의 권리는 각 권리자에게 귀속됩니다.
- 본 저장소는 해당 제3자 IP에 대해 어떠한 라이선스나 사용 허가를 부여하지 않습니다.
- 저장소의 구현 코드와 문서는 개인 사이드 프로젝트 차원의 기록물이지만, 캐릭터/IP 자체에 대한 권리를 주장하지 않습니다.
