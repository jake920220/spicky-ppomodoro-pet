# 아키텍처 규칙

## 관심사 분리

프로젝트는 아래 관심사를 분리한다.

- Tauri 창/쉘 동작
- UI 렌더링
- 타이머 도메인 로직
- 캐릭터 시각 상태 전환
- 오디오 재생
- 에셋 경로 관리

## 권장 모듈 구조

- src/app
- src/features/timer
- src/features/spiky-state
- src/features/audio
- src/shared/assets
- src/shared/types

## 구조 원칙

- UI 컴포넌트가 타이머 비즈니스 로직을 직접 소유하지 않도록 한다.
- 타이머 상태가 시각 상태를 유도해야 하며, 반대로 시각 상태가 타이머 로직을 지배하면 안 된다.
- 오디오는 명확한 인터페이스를 통해 타이머 완료 이벤트에 반응해야 한다.
- 에셋 식별자와 경로는 중앙집중적으로 관리한다.

## 복잡도 원칙

- 과한 확장성 설계는 피한다.
- 초반부터 무거운 플러그인 구조를 넣지 않는다.
- 단순한 모듈과 명시적인 데이터 흐름을 선호한다.

## 구현 원칙

- 우선 동작하는 최소 기능을 만든다.
- 리팩터링은 기능 검증 이후에 한다.
- 변경은 작고 검토 가능한 단위로 유지한다.
